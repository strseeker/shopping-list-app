# PRD Step 1: 문제 정의 및 현황 분석

> 목표: 웹 포트폴리오 방문자의 체류 시간을 현재의 2배로 늘리기

---

## 1. 목표 설정

| 항목 | 내용 |
|------|------|
| 목표 | 평균 세션 지속 시간 2배 증가 |
| 기간 | 2개월 이내 |
| 측정 도구 | Google Analytics 4, Microsoft Clarity |

---

## 2. 체류 시간 저하의 근본 원인 분석

### A. 콘텐츠 측면
- 단순 프로젝트 목록 나열 → 스크롤 한 번에 끝남
- 텍스트 위주의 정적인 정보 → 읽는 동기 부족
- 업데이트 없는 완성된 사이트 → 재방문 이유 없음
- 스토리가 없는 기술 나열 → 감정적 연결 부재

### B. UX/기술 측면
- 다음 액션 유도 요소 부족 (CTA 부재)
- 페이지 간 연결이 약해 탐색 중단
- 모바일 UX 미흡

### C. 참여 메커니즘 부재
- 인터랙티브 요소 없음
- 데모, 라이브 프리뷰 없음
- 방문자가 '해볼 수 있는' 것이 없음

### D. 재방문 유인 없음
- 블로그/아티클 없음
- 주기적 업데이트 없음

---

## 3. 핵심 KPI 정의

### Primary KPI
- **평균 세션 지속 시간**: 현재 X → 목표 2X
- **페이지뷰/세션**: 현재 → 목표 +50%

### Secondary KPI
- 이탈률(Bounce Rate): 10~15% 감소 목표
- 스크롤 깊이: 70% 이상 도달 비율 증가
- 프로젝트 페이지 클릭률: +30%
- 재방문율: +20%

---

|------|------|------|

| Microsoft Clarity | 히트맵 + 세션 녹화 | 무료 |
| Hotjar | 스크롤 맵, UX 분석 | 무료(기본) |
