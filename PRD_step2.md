# PRD Step 2: 전략 수립 및 우선순위 매트릭스

> 이전 단계: PRD_step1.md (문제 정의 및 현황 분석)

---

## 1. 핵심 전략 5가지

### 전략 1: 콘텐츠 깊이 강화 (케이스 스터디)
각 프로젝트를 단순 나열이 아닌 스토리로 재구성

```
프로젝트 페이지 구조:
[히어로 섹션] 프로젝트명, 한 줄 요약, 사용 기술
[배경]        왜 만들었나? 어떤 문제를 해결하나?
[과정]        기획 → 설계 → 개발 → 배포 타임라인
[핵심 기능]   스크린샷 + 설명
[기술적 도전] 어려웠던 점과 해결 방법
[결과 & 회고] 성과, 배운 점, 개선 사항
[다음 프로젝트 →] 내비게이션
```
- 예상 읽기 시간: 5~8분/프로젝트

### 전략 2: 인터랙티브 요소 추가
- 라이브 데모 임베드 (iframe 또는 CodeSandbox)
- 인터랙티브 기술 스택 시각화 + 호버 툴팁
- 스크롤 기반 애니메이션 (Framer Motion / GSAP)
- 프로젝트 필터링 (언어별, 카테고리별)

### 전략 3: 내비게이션 흐름 최적화
- "다음 프로젝트 보기 →" 버튼으로 프로젝트 간 이동 유도
- 관련 프로젝트 추천 섹션 ("You might also like")
- TOC(목차) 사이드바로 긴 페이지 탐색 용이
- 읽기 진행률 바 (상단 고정)

### 전략 4: 방문자 유형별 맞춤 경로
- 랜딩 페이지에 "I am a..." 선택 버튼
  * 채용 담당자 → 주요 프로젝트 + 이력서 다운로드
  * 동료 개발자 → 기술 블로그 + 오픈소스 기여
  * 클라이언트 → 완성된 결과물 위주 쇼케이스

### 전략 5: 갱신 가능한 콘텐츠 구조
- 기술 블로그/아티클 섹션 (월 1~2개 포스트)
- 현재 진행 중인 프로젝트 섹션 ("Now Building")
- GitHub 활동 통합, 독서/학습 로그

## 3. 3단계 로드맵

### Phase 1: Quick Win (1~2주)
1. 프로젝트 케이스 스터디 작성
2. "다음 프로젝트 →" 내비게이션 버튼
3. 예상 읽기 시간 표시
4. TOC 사이드바

**예상 효과: +40~60% 체류 시간**

### Phase 2: 인터랙션 강화 (2~4주)
5. 읽기 진행률 바
6. 기술 스택 인터랙티브 호버
7. 스크롤 애니메이션

**예상 효과: 추가 +20%**

### Phase 3: 콘텐츠 생태계 구축 (1~2개월)
8. 블로그 섹션 (월 2개 포스트)
9. 라이브 데모 임베드
10. 방문자 유형별 경로

**예상 효과: 추가 +30~50%**

## 4. 위험 요소 및 완화 방안

| 위험 | 완화 방안 |
|------|-----------|
| 콘텐츠 과다로 압도감 | 섹션별 접기/펼치기, 진행 표시 바 |
| 인터랙션 추가로 로딩 속도 저하 | 레이지 로딩, 이미지 최적화 |
| 콘텐츠 업데이트 부담 | 템플릿화, 간단한 CMS 도입 |
